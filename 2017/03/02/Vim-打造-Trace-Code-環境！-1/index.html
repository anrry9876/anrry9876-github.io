引言

雖然，我個人比較喜歡 Sublime Text 的環境，但是有一天心血來潮，還是嘗試了自己比較不熟的Vim，這支老牌的編輯器。我個人非常討厭 Windows ，原因其實也沒什麼，就是不喜歡僅此而已，所以我想要在 Mac os 上面自己打造一個可以 TraceCode 的環境，網路上或許有許多現成的 IDE可以使用，但秉持著我喜歡動手自己來的精神，決定自己打造一款，適合自己的 Vim 來提升工作效率。 Let’s Go!

# 選擇自己喜歡的 Vim 樣式

貼心小提醒 以下會貼上 Mac os 與 Ubuntu 的版本，供大家使用！使用不同的程式語言撰寫程式的時候
人們會希望某一些程式的關鍵字或是函式可以設定成高亮狀態 highlight
也就是用不同的顏色來顯示這些關鍵字或者是函示
但是
人生就是這個但是！！！
不同的程式語言，就會有不一樣的關鍵字
為了可以產生不同語言的高亮效果
我們可以設定不同的佈景主題
這樣的一個功能
在 Vim 的樣式有一個標準的名字叫做 Color Scheme
很貼心的，在 Vim 的官方網站上，有提供很多預設的 Color Scheme 可以任君挑選！

挑選完自己喜歡的 colorscheme 之後
先到家目錄下建立 .vim 資料夾，然後再建立colors資料夾！

下載下來的 colorscheme 是以 .vim 做為副檔名的檔案，
下載完成後複製至 ~/.vim/colors 這個資料夾內就可以了！

syntax enable
colorscheme xxx

$ git clone https://github.com/VundleVim/Vundle.vim.git ~/.vim/bundle/Vundle.v

:PluginInstall

“————–
“ Color Scheme
“————–

syntax enable
colorscheme spring-night
set mouse=nv

“—————-
“ 編輯器基本設定
“—————-
“ 默認顯示行號
set nu
“設置（軟）製表符寬度為4：

set tabstop=4
set softtabstop=4

“ 設置自動縮進：即每行的縮進值與上一行相等；使用 noautoindent 取消設置：
set autoindent
“ 設置（自動）縮進的空格數為4
set shiftwidth=4
“ 設置 使用 C/C++ 語言的自動縮進方式：
set cindent
“ 智能縮進
set smartindent
“在狀態欄顯示正在輸入的命令
set showcmd
“搜索的時候即時顯示結果
set incsearch
“ 高亮搜索結果
set hlsearch
“ 禁止循環查找
set nowrapscan
“ 設置匹配模式，顯示匹配的括號
set showmatch
“ 智能補全
set completeopt=longest,menu
“ 設置歷史記錄為100條
set history=100

“標尺，用於顯示光標位置的行號和列號，逗號分隔。每個窗口都有自己的標尺。如果窗口有狀態行，標尺在那裡顯示。否則，它顯示在屏幕的最後一行上。”

set ruler
set ai
set cursorline

“======================================================
“ Start vundle
“======================================================

set nocompatible “ be iMproved, required
filetype off “ required

“ set the runtime path to include Vundle and initialize
set rtp+=~/.vim/bundle/Vundle.vim
call vundle#begin()

“ alternatively, pass a path where Vundle should install plugins
“call vundle#begin(‘~/some/path/here’)
“ let Vundle manage Vundle, required

Plugin ‘VundleVim/Vundle.vim’

“============================================
“ Write your plugins here
“============================================
Plugin ‘Yggdroot/indentLine’
Plugin ‘ntpeters/vim-better-whitespace’
Plugin ‘vim-airline/vim-airline’
Plugin ‘tpope/vim-fugitive’
Plugin ‘chazy/cscope_maps’
Plugin ‘vim-scripts/taglist.vim’
Plugin ‘scrooloose/nerdtree’
Plugin ‘wesleyche/SrcExpl’
Plugin ‘wesleyche/Trinity’
Plugin ‘majutsushi/tagbar’

“================================================
“ Run vundle
“================================================
“ All of your Plugins must be added before the following line
call vundle#end() “ required
filetype plugin indent on “ required
“ To ignore plugin indent changes, instead use:
“filetype plugin on

“ Brief help

“ :PluginSearch foo - searches for foo; append ! to refresh local cache
“ :PluginClean - confirms removal of unused plugins; append ! to auto-approve removal
“ see :h vundle for more details or wiki for FAQ
“ Put your non-Plugin stuff after this line

nmap :TrinityToggleAll

“ “ Open and close the Source Explorer separately
nmap :TrinityToggleSourceExplorer

nmap :TrinityToggleTagList
“

# 結語

這樣就可以免費在linux base 的系統上追蹤code 了，且速度快，滿無痛的
最大的成本應該是學 vim的熱鍵！但是對於熟悉vim的人來說，上手是非常快速的！
